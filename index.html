HEXAPOD
RESCUE ROBOT

An open-source research framework for building a 6-legged autonomous rescue robot capable of stair navigation, patient transport, and unstructured terrain traversal. Based on original concept designs for emergency stair evacuation. Built upon prior research in aircraft vibration isolation for life-critical human transport.

CORE MODULES TO BUILD
01 — Locomotion Engine
Gait planner with tripod, wave, and adaptive stair gaits. Foot trajectory generation using spline interpolation. Contact detection and foothold optimization.
Python / C++ / ROS2
02 — Inverse Kinematics
Per-leg IK solver for the R-R-R-P-R-R chain. Whole-body kinematics with patient capsule stabilization constraint. Real-time joint trajectory computation.
NumPy / Pinocchio
03 — Perception Stack
Stereo camera + LiDAR fusion for terrain mapping. Stair detection via plane segmentation. 3D foothold candidate generation on unstructured terrain.
Open3D / PointNet
04 — RL Policy Training
Isaac Gym / MuJoCo environment for locomotion policy learning. PPO agent with privileged critic. Reward shaping for stair descent and patient safety constraints.
Isaac Lab / MuJoCo
05 — Patient Safety System
IMU-based capsule orientation monitoring derived from aircraft isolator research. Active stabilization to keep patient horizontal ±5°. Emergency stop and safe settle behavior on fault detection.
IMU / PID / ROS2
06 — Tele-op + Autonomy
Handheld operator controller with live video feed. Shared-autonomy mode where operator sets waypoints, robot handles foot placement. Full autonomous fallback.
WebRTC / ROS2

PRIOR ART & REJECTED RESEARCH

How aircraft isolator research for infant transport became the missing piece in rescue robotics — and how the Freenove hexapod platform validated the locomotion baseline used to build HEXEVAC from the ground up.

FREENOVE HEXAPOD — REFERENCE TECHNICAL DRAWINGS

The following technical drawings are adapted from the Freenove Hexapod Robot Kit documentation, used here as reference material under CC BY-NC-SA 3.0. Attribution: Freenove (freenove.com). These drawings informed HEXEVAC's leg joint configuration and servo layout. HEXEVAC extends this locomotion foundation with life-critical patient stabilization systems derived from original aircraft isolator research.

FIG 1 — HEXAPOD FULL ASSEMBLY (TOP VIEW) FREENOVE REF · CC BY-NC-SA 3.0
Reference basis for HEXEVAC locomotion: The Freenove hexapod uses 18 MG90S servos arranged as 3 joints per leg across 6 legs. The alternating tripod gait — where Legs 1, 3, 5 swing while 2, 4, 6 support — provides static stability at all times. HEXEVAC retains this proven symmetrical configuration while scaling to quasi-direct-drive actuators (T-Motor AK80-9) capable of supporting a patient load. The joint circle markers above correspond to the coxa, femur, and tibia servo positions. In HEXEVAC's R-R-R-P-R-R chain, this 3-joint configuration becomes the first three revolute joints, with a prismatic actuator and two ankle joints added for terrain compliance and capsule stabilization.
FIG 2 — SINGLE LEG ANATOMY (SIDE VIEW) FREENOVE REF · 3-DOF
3-DOF Freenove leg → 6-DOF HEXEVAC leg: Each Freenove leg uses 3 MG90S servos — coxa (hip yaw ±60°), femur (hip pitch -30° to +80°), and tibia (knee pitch). The tibia directly terminates at the foot contact point. HEXEVAC takes this proven geometry and extends it: between the knee and foot, a prismatic linear actuator is added for terrain compliance (absorbing step impact), followed by two ankle joints (pitch + roll) to actively keep the foot flat on irregular surfaces. This directly mirrors the gimbal isolator mechanism from the aircraft infant transport system — same principle, rescaled for a 90kg patient load.
FIG 3 — SERVO CONTROL TOPOLOGY FREENOVE REF · 18-CH PWM
PWM servo control topology → CAN bus architecture: Freenove routes 18 PWM signals through a PCA9685 I²C expander to an Arduino Mega at 50Hz update rate. This is sufficient for a hobby hexapod but creates a control bottleneck for real-time stabilization. HEXEVAC replaces this entirely with a CAN bus topology — one Moteus r4.11 controller per joint, running at 1kHz with torque feedback. The same wiring concept (legs grouped by tripod: left group L4/L5/L6 and right group L1/L2/L3) is preserved in HEXEVAC's CAN network topology, making the channel mapping directly comparable for reference during firmware development.
FIG 4 — SERVO CALIBRATION PROFILE FREENOVE REF · PULSE WIDTH MAPPING
Servo calibration → actuator characterization pipeline: The Freenove calibration process establishes the linear mapping from PWM pulse width (500–2000μs) to joint angle (-90° to +90°). The ±2° tolerance band shown was measured during prototype testing on stairs and flat surfaces. This calibration data is directly applicable to HEXEVAC's motor characterization pipeline — the same concept of commanded-vs-actual joint angle mapping applies to torque-controlled quasi-direct-drive motors. HEXEVAC replaces PWM open-loop control with torque + position closed-loop control via Moteus, but the calibration methodology (neutral offset, per-joint trim) is identical.
FIG 5 — TRIPOD GAIT PHASE DIAGRAM FREENOVE REF · ADAPTED FOR STAIR DESCENT
Tripod gait foundation for stair descent adaptation: The Freenove tripod gait divides 6 legs into two alternating groups of 3. At any moment, one tripod is in stance (supporting the robot) while the other swings forward. This guarantees a statically stable support polygon at all times — critical for carrying a patient. HEXEVAC's stair descent gait builds directly on this foundation: during each swing phase, the 3 swinging legs are targeted to the leading edge of the next step down, with foot placement computed from LiDAR stair detection. The stance tripod simultaneously lowers the body with controlled deceleration, keeping the patient capsule tilt within ±5°. This is the exact same physics as the aircraft isolator's active gimbal — maintaining a horizontal platform during a controlled descent event.

NESTARA
MODULAR NEONATAL TRANSPORT ISOLATION SYSTEM
A full clinical design engineering project developed through field immersion at UCAN/Lurie Children's Hospital and Boston MedFlight. Nestara is a 3D-printed auxetic metamaterial lattice bed insert designed to slide into existing neonatal isolettes — providing localized vibration isolation and passive thermal buffering without adding complexity, power requirements, or disrupting clinical workflows. The core research question: whether a BCC (Body-Centered Cubic) lattice structure adapted from aerospace can be tuned for infant physiology at accessible manufacturing tolerances — or whether simpler materials (open-cell foam, established elastomers) outperform complex geometry at this scale. Testing showed amplified vibration in some conditions, indicating over-engineering may be the problem.

// ACTUAL PROTOTYPE PHOTOGRAPHS — NESTARA V1 TESTING
PHOTO 1 — BCC LATTICE FLAT (TOP) PASSIVE ISO LAYER
Auxetic BCC lattice — the core invention. Diamond-cell geometry adapted from aerospace BCC structures, scaled ×5 and tuned for the 0.1–2Hz neonatal WBV threshold. Negative Poisson's ratio means cells expand laterally under vertical compression — distributing impact load rather than concentrating it. nTopology-generated for high-resolution cell uniformity. Open finding: FDM printing resolution may be insufficient to realize the intended geometry — cell wall tolerances at consumer printer settings can shift resonant frequency into the harmful range rather than away from it. Higher-resolution printing or simpler geometry (reticulated foam, TPU sheet) may be the correct next step.
PHOTO 2 — INSTRUMENTED PROTOTYPE ACTIVE SENSING LAYER
Wired for data acquisition — stair + flat transport test configuration. Sensor/actuator cables routed through the diamond-cell channels. This was the configuration used in the calibrated shock protocol: 2× 50lb weights as surrogate load, accelerometer-equipped phone at lattice base. The routing-inside-lattice design protects signal wires from the same vibration being measured — a key finding for HEXEVAC capsule harness design.
PHOTO 3 — CORRUGATED BASE SIDE VIEW SPRING COMPLIANCE LAYER
Accordion-fold corrugated base — primary vertical travel. V-groove geometry provides 8–15mm progressive-stiffness stroke: soft initial response with hardening as compression deepens. Scalloped bottom profile creates stress-relief notches for cyclic durability (500+ transport cycles). This layer maps directly to HEXEVAC's prismatic joint (the P in R-R-R-P-R-R) — same physics, different scale.
PHOTO 4 — SUBSTRATE COUPLING TEST SURFACE INTEGRATION
Lattice on flexible substrate — isolette surface coupling validation. Tests how the lattice performs when the base surface (isolette mattress or stretcher fabric) is itself deformable. Finding: auxetic lattice outperforms foam at decoupling patient layer from base-surface deformation, especially in the 8–20Hz ambulance/helicopter vibration range. Critical data point for HEXEVAC capsule shell-to-robot-body interface specification.
PHOTO 5 — FIN ARRAY CLOSE-UP TUNED MASS DAMPERS
Cantilevered fin array — broadband tuned mass dampers. Individual fin height, width, and spacing set each fin's resonant frequency. Two instrumented fins (blue/cyan) carry strain gauges. Staggered fin geometry gives broadband attenuation across 5–50Hz by spreading natural frequencies across the array. Shore 90A PLA composite — similar hardness to a caster wheel. Provides passive damping floor even if active control is disabled.
PHOTO 6 — LAB BENCH SETUP DATA ACQUISITION RIG
NICU room-to-room test configuration. Lattice on padded surface, laptop for real-time accelerometer logging via phone dashboard. This is the Tier 1 test in the calibration plan: baseline controlled-environment performance. Results (shown in IDEP paper): test runs showed mixed results — some runs showed amplified vibration vs baseline, indicating the BCC geometry at FDM resolution may be hitting resonance bands rather than attenuating. This is the open research question: whether geometry refinement, different cell types, or switching to established foam materials is the right path forward.
PHOTO 7 — STACK ANGLE VIEW 3-LAYER ARCHITECTURE
Full stack visible: lattice top + corrugated base edge. Rated for 500+ transport cycles. The fan/wedge corrugation geometry provides the same non-linear stiffness profile as aerospace landing pad bumpers — essential for the unpredictable impact profiles of real hospital transport (elevator gaps, door thresholds, ambulance loading ramps). For HEXEVAC: this multi-layer approach informs the capsule mount spec — passive layers first, then active IMU correction.
PHOTO 8 — PATIENT INTERFACE PAD TOP CONTACT LAYER
Viscoelastic patient contact pad — final isolation stage. Micro-perforated surface for breathability. Cuts off frequencies above 50Hz where lattice efficiency drops. Potential color-change thermal layer option (visual temp feedback to nurses, no extra probes needed). This maps directly to HEXEVAC capsule interior lining: soft interface → rigid structural frame → isolator mount stack. The color-change thermal concept translates to patient monitoring HUD in HEXEVAC's operator dashboard.

# hexevac/control/kinematics.py — IK solver for 1 leg import numpy as np class HexevacLegIK: def __init__(self, L1=0.15, L2=0.22, L3=0.18): self.L1 = L1 # coxa length (m) self.L2 = L2 # femur length self.L3 = L3 # tibia length def solve(self, foot_pos: np.ndarray) -> np.ndarray: x, y, z = foot_pos q1 = np.arctan2(y, x) r = np.sqrt(x**2 + y**2) - self.L1 d = np.sqrt(r**2 + z**2) cos_q3 = (d**2 - self.L2**2 - self.L3**2) / (2*self.L2*self.L3) q3 = np.arccos(np.clip(cos_q3, -1, 1)) alpha = np.arctan2(-z, r) beta = np.arctan2(self.L3*np.sin(q3), self.L2 + self.L3*np.cos(q3)) q2 = alpha - beta q5 = -(q2 + q3) q6 = 0.0 q4 = 0.0 # terrain-adaptive extension return np.array([q1, q2, q3, q4, q5, q6])

# ai/locomotion_policy/train_hexevac.py class HexevacStairEnv(DirectRLEnv): def _compute_rewards(self, obs, actions): rew = 1.0 * self._rew_forward_progress() rew += 2.0 * self._rew_stair_descent_success() rew -= 5.0 * self._pen_patient_tilt_exceeded(0.087) # >5° rew -= 0.5 * self._pen_action_rate(actions) rew -= 10.0 * self._pen_fall() rew -= 1.0 * self._pen_joint_limit_violation() return rew TRAIN_CFG = { "algorithm": "PPO", "num_envs": 4096, "curriculum": "flat → ramps → stairs → rubble", }

# ros2_ws/hexevac_perception/stair_detector.py class StairDetector: def detect(self, cloud: o3d.geometry.PointCloud): ds = cloud.voxel_down_sample(0.02) planes = self._segment_planes(ds, max_planes=10) steps = [p for p in planes if self._is_horizontal(p) and 0.1 < p.z_mean < 2.5] steps.sort(key=lambda p: p.z_mean) return [(s.z_mean, s.x_far) for s in steps]

# ros2_ws/hexevac_control/patient_safety.py class PatientSafetyMonitor: MAX_TILT_DEG = 5.0 CRITICAL_TILT = 12.0 def update(self, capsule_imu): roll = np.degrees(capsule_imu.roll) pitch = np.degrees(capsule_imu.pitch) tilt = np.sqrt(roll**2 + pitch**2) if tilt > self.CRITICAL_TILT: self._trigger_emergency_settle() return if tilt > self.MAX_TILT_DEG: correction = self._compute_body_correction(roll, pitch) self.body_pose_pub.publish(correction)
